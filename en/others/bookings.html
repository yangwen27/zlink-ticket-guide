Latest version of the booking system operation manual

Login system

System Management - Log View (all records left on the operating system)

User Management

  1. Open System Management
  2. Open User Management

  3. Click New

    1. Account: The account for logging in to the system (English, numbers, symbols)
    2. Password: Set the password
    3. Name: The name of the user of this account
    4. Role: What role is this account authorized to be
    5. User group: What is the user group of this account

  4. Submit after editing

Role management

Role management: Role management means that the user is assigned to this role. In this role, the visible pages and operation permissions are set. After logging in to the system through this user, the user can see the pages and operable permission functions that you have authorized to display.

  1. Open System Management
  2. Open Role Management

  3. New

    1. Role number: You can number it at will but it cannot be repeated (numbers, English, symbols), it is only for recording purposes;
    2. Role name: What is the name of this new role? (For example, if you can operate group reservations, you can name it: Group Reservation);
    3. Remarks: Remarks, instructions, etc. for this new role.

  4. Set the page permissions of the role

    1. Find the corresponding role - page permissions; this is to set the pages that the role can see and the operations that can be performed after logging in to the system; check it to be visible and operable; if it is not checked, it is not visible and not operable
    2. Submit after editing

User group management

Create user group

  1. User group number: number setting, for marking function;
  2. Group name: What is the name of this new group?
  3. Remarks: Special remarks for this group;
  4. Submit;

User group data permission settings

Find the corresponding group - click data permissions

  1. Ticket distributor: The reservation system does not use this function, it can be relaxed, and there is no other impact
  2. Ticket seller: All permissions need to be set, which is for ticket sales and viewing data
  3. Ticket checking equipment: All permissions need to be set, which is required for viewing ticket checking data
  4. Submit

Set the user group of the user

Menu management settings

Menu management is the setting of the left menu bar. We have already set it all up, so we don't need to move it. If you don't know how to set it, you can contact customer service and let our customer service guide you. After modification, you need to refresh the menu and log in again to take effect

Data permission management

Data permissions: Data permissions correspond to data, including distributor data, ticket checking data, and ticket seller data. You can only see the corresponding data after setting the permissions.

  1. Click the left menu - System Management;
  2. Click the left menu - Data Permissions;

  3. Click New;

    1. Permission category: generally set user group permissions,
    2. Permission name: What is the name; (for example, the platform administrator needs to view all data permissions, so give him all permissions);
    3. All permissions: Yes/No;
    4. User group: select a user group for authorization;

  4. Submit;

New personal reservation ticket

  1. New personal reservation ticket
    1. Basic parameters
      1. Name: The name of the ticket (the name of the personal reservation ticket.
      2. Ticket number: The number of each ticket is different
      3. For use on the same day: This function is not applicable to ticket reservations
      4. Expiration method: This function is not applicable to ticket reservations
      5. Validity period: The validity period can be determined according to the use time of the ticket, without restriction
